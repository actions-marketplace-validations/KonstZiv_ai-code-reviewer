# Спрінт 1: MVP Ревʼювер коду - Перевірка інструментарію та перший реліз

**Мета спрінту:** Створити найпростіший робочий ревʼювер коду для перевірки всього процесу розробки та інструментарію

**Тривалість:** 1 спрінт (гнучко)
**Статус:** 🎯 Готовий до старту
**Тип:** Фундаментальний спрінт
**Пріоритет:** Критичний (блокує всю іншу роботу)

## 🎯 Цілі спрінту

### Основна мета
Побудувати найпростіший можливий робочий ревʼювер коду який:
- Аналізує GitHub MR на критичні вразливості
- Перевіряє чи зміни коду відповідають опису завдання
- Публікує ревʼю як коментар до MR
- Використовує безкоштовний Google Gemini API

### Додаткова мета (найважливіша!)
**Перевірити весь процес розробки:**
- ✅ Інструменти якості коду (ruff, mypy)
- ✅ Тестовий pipeline (pytest + покриття)
- ✅ Pre-commit хуки
- ✅ CI/CD pipeline (тести, документація, реліз)
- ✅ Багатомовна документація
- ✅ Публікація в PyPI
- ✅ Основи інтеграції з LLM

## 🏗️ Архітектура

### Високорівневий потік
```
GitHub MR створено/оновлено
    ↓
CI запущено
    ↓
Перевірки якості (ruff, mypy)
    ↓
Тести (pytest)
    ↓
AI Reviewer Job
    ↓
[Отримати дані MR] → [Отримати повʼязане завдання] → [Побудувати контекст]
    ↓
[Надіслати до Gemini] → [Парсити відповідь]
    ↓
[Опублікувати коментар ревʼю]
```

### Компоненти для створення

```python
# Мінімальна структура
src/ai_reviewer/
├── core/
│   ├── models.py          # Pydantic моделі для MR, Task, Review
│   └── config.py          # Керування конфігурацією
│
├── integrations/
│   ├── github.py          # Отримання даних MR
│   └── gemini.py          # Google Gemini клієнт
│
├── reviewer.py            # Основна логіка ревʼю
└── cli.py                 # Точка входу: ai-review команда
```

### Моделі даних

```python
# Спрощені моделі
class MergeRequest:
    title: str
    description: str
    comments: List[Comment]
    changes: List[FileChange]

class LinkedTask:
    title: str
    description: str

class ReviewContext:
    mr: MergeRequest
    task: LinkedTask | None

class ReviewResult:
    has_critical_vulnerabilities: bool
    vulnerabilities: List[str]
    matches_task: bool | None  # None = неможливо визначити
    reasoning: str
```

## 🚀 CI/CD Pipeline

### При Push/PR (tests.yml)
```yaml
jobs:
  quality:
    - ruff check
    - ruff format --check
    - mypy src/

  test:
    - pytest --cov=ai_reviewer
    - Завантажити покриття

  ai-review:  # НОВЕ
    needs: [quality, test]
    - Запустити ai-review на PR
```

### При тегу (release.yml)
```yaml
# Тег: v0.1.0
jobs:
  test: [запустити всі тести]
  build: [зібрати пакет]
  publish-pypi: [опублікувати в PyPI]
  github-release: [створити GitHub реліз]
  deploy-docs: [задеплоїти багатомовну документацію]
```

### Деплой документації (docs.yml)
```yaml
# При push до main
- Зібрати всі мовні варіанти
- Задеплоїти на GitHub Pages
```
